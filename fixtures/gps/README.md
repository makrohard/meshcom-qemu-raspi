# Synthetic GPS NMEA fixtures

These files are **synthetic test vectors** for the QEMU GPS fixture tests
(`scripts/test-gps.sh`, fed via `scripts/gps-relay.py --mode fixture`). They contain
**deliberately fictional coordinates** — a round point at **48°00.0000′N 012°00.0000′E**
— and are **not** captured from any real receiver. No live position is stored here.

All sentences carry correctly computed NMEA XOR checksums (except `bad_checksum.nmea`,
which is intentionally corrupt). Checksums were generated by `python3` XOR over the
characters between `$` and `*`.

| File | Purpose |
|------|---------|
| `valid_fix.nmea`   | Valid fixed position: RMC status `A`, GGA sats=7 (>5), HDOP=1.2 (<6.0) → drives a normal MeshCom fix. |
| `no_fix.nmea`      | Valid checksums but no position (RMC `V`, GGA quality 0, empty lat/lon) → must never become a fix. |
| `bad_checksum.nmea`| Well-formed printable shape with a wrong checksum → parser must stay safe (no crash, no false fix). |
| `stale.nmea`       | A short burst of valid sentences; the relay stops at EOF (`--once`) → the fix must age out (`age_ms ≥ 5000`). |
| `short_track.nmea` | Three distinct epochs moving north → position must update, not lock to the first point. |

The relay paces these at ~1 sentence/sec; an RMC+GGA pair is one 1 Hz epoch.
